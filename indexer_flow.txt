索引构建流程图
===============

data/ 目录
  ├── cell_biology.txt        ┐
  └── molecular_biology.txt   ┘
         │
         ▼
┌─────────────────────┐
│  第 1 步：加载文档    │  load_documents()
│  扫描 data/ 目录      │  → [{"source": "xxx.txt", "content": "..."}]
│  读取 .txt/.md/.docx/.pdf │
└─────────────────────┘
         │
         ▼
┌─────────────────────┐
│  第 2 步：文本切块    │  re.split() + 循环
│  按空行分割成段落     │
│  短段落 → 合并       │  例如：
│  长段落 → 强制切割    │  "DNA复制需要...（500字）"
│  保留 64 字符重叠     │  → chunk_0: "DNA复制需要..." (512字符)
└─────────────────────┘  → chunk_1: "...解旋酶打开双..." (512字符, 和上一块重叠64字符)
         │                → chunk_2: "...引物酶合成RNA..." (剩余)
         ▼
┌─────────────────────┐
│  第 3 步：编码向量    │  BGEM3FlagModel.encode()
│  bge-m3 模型         │
│  输入：文本           │  例如：
│  输出：1024维向量     │  "DNA复制需要解旋酶"
│  batch_size=8        │  → [0.12, -0.34, 0.56, ..., 0.78]  (1024个浮点数)
└─────────────────────┘
         │
         ▼
┌─────────────────────┐
│  第 4 步：存入 ChromaDB │  collection.add()
│  向量数据库           │
│  存储：向量+文本+元数据 │  ChromaDB 会自动建索引
│  持久化到磁盘         │  查询时用余弦相似度找最近的
└─────────────────────┘
         │
         ▼
    chroma_db/ 目录（.gitignore 忽略）
    ├── biograph/（集合数据）
    └── ...
```

查询流程（retriever.py）：
  用户提问 "DNA复制需要哪些酶？"
         │
         ▼
  bge-m3 编码 → [0.23, -0.11, 0.45, ...]  (query 向量)
         │
         ▼
  ChromaDB 查询 → 找到最相似的 20 个 chunk
         │
         ▼
  Reranker 精排 → 从 20 个中选出最好的 3 个
         │
         ▼
  交给 LLM 生成回答
```
